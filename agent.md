# Agent Notes

## Current Objective

Run Stackelberg CodePo experiments with a paper-faithful strict alternating order:

1. sample follower/coder context trajectories
2. build follower preference pairs
3. train follower/coder adapter
4. resample leader trajectories with the updated follower
5. build and clean leader preference pairs
6. train leader/planner adapter
7. evaluate the joint leader-follower system

Do not use the old synchronous/common-experience-pool runner for conclusions about the strict algorithm.

## Conservative Guarded Defaults

The guarded run intentionally starts smaller than the failed aggressive run:

```text
sample_limit = 32
sample_max_rounds = 2
planner_temperatures = [0.2, 0.8]
follower_temperatures = [0.2, 0.8]
lora_rank = 16
lora_alpha = 32
learning_rate = 3e-6
leader_train_steps = 80
follower_train_steps = 120
train_max_length = 1024
parallel GPUs = 0 2 by default
```

This is meant to prevent the previous failure mode: a large serial run where all generated code became syntax errors, no early stop fired, and almost no useful preference pairs were created.

## Health Gates

The strict pipeline now supports `health_gates` in the config. When enabled, it stops early before wasting training time.

Current guarded checks:

```text
01_follower_context_sampling:
  trajectories >= 32
  syntax_error_rate <= 0.35
  positive_pass_rate_fraction >= 0.05
  avg_rounds <= 1.95

02_follower_preference_data:
  wpo >= 8
  preferences >= 8
  candidates >= 16

04_leader_sampling_after_follower:
  trajectories >= 32
  preferences >= 4
  syntax_error_rate <= 0.35
  positive_pass_rate_fraction >= 0.05
  avg_rounds <= 1.95

08_joint_eval:
  num_tasks >= 8
  avg_assert_pass_rate_final >= 0.20
```

If a gate fails, inspect `outputs/strict_guarded_train32_round2_iter0/manifest.json`, especially `health_gates`.

## Monitoring

Use these files instead of watching terminal output:

```bash
tail -n 80 outputs/verification_logs/strict_guarded_train32_round2_iter0_master.log
cat outputs/verification_logs/strict_guarded_train32_round2_iter0.heartbeat
cat outputs/verification_logs/strict_guarded_train32_round2_iter0.pid
```

Check active processes:

```bash
ps -eo pid,ppid,etime,cmd | grep -E 'strict_guarded|strict-alternating-smoke|leader_sampling|follower_sampling|weighted_dpo|role_eval' | grep -v grep
```

Summarize results:

```bash
python - <<'PY'
import json
from pathlib import Path
p = Path("outputs/strict_guarded_train32_round2_iter0/manifest.json")
if p.exists():
    m = json.loads(p.read_text())
    print("counts:", m.get("counts"))
    print("health_gates:", m.get("health_gates"))
    print("eval:", (m.get("summaries") or {}).get("eval"))
PY
```

## Cleanup Policy

Do not delete experiment outputs containing manifests or summaries unless explicitly asked. It is acceptable to archive noisy verification logs into:

```bash
outputs/verification_logs/archive_YYYYmmdd_HHMMSS
```

## Important Reminder

The previous GPU protection/occupancy settings were useful while sleeping. Remember to release or disable any temporary GPU reservation/locking setup after confirming no long experiment needs it.
